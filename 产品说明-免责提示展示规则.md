# 产品说明：免责提示展示规则

## 1. 首页（入口页）
- **展示要求**：不展示任何免责提示内容
- **设计原则**：保持页面简洁，避免干扰用户核心操作

## 2. 对应业务页面
- **适用场景**：涉及风险、协议或重要告知的页面（如：记账确认页、工资结算页、招聘发布页、个人信息授权页等）
- **展示位置**：通常放在页面底部、按钮上方，或作为独立的"用户须知"模块
- **展示形式**：
  - 可折叠/展开的文本块，默认折叠，用户可点击查看详情
  - 或作为复选框前的说明文字，用户勾选即表示已阅读并同意

## 3. 内容原则
- **相关性**：每个页面的免责提示必须与该页面的业务场景强相关，避免通用化、无意义的条款堆砌
- **可读性**：语言通俗易懂，避免法律术语，确保一线员工能快速理解

## 4. 实施建议
- 开发团队应根据具体页面的业务场景，选择合适的展示形式
- 确保免责提示内容与页面功能紧密相关，提供有针对性的风险提示
- 在用户进行关键操作前，确保用户已阅读并理解相关免责提示
- 定期 review 免责提示内容，确保其准确性和有效性
